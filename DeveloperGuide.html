Welcome

The biggest component is our github code, this is the source of our library. Our interface uses this source code in its calculation. Our Pypi project is created from our github code.

Making Changes

Your local machine is where you are going to make most of the edits and changes. If our interface or library produces inaccurate results, the problem is in our python library. For this case you would want to "Maintain the python code". If there is a problem with how our interface looks and behaves, you are going to want to "Maintain the interface's code".

Maintain The Python Code

First thing you want to do is gain access to our github repo, contact Scott Turro for access. Our package is entirely in python so you will need some IDE or text editor to edit the code. I used pycharm and really liked the debugging and git control features it had.

  1. Use your computer to change and test the code

    2. Once you have access, you can clone the repo to your computer using the git clone command and make your changes. You should create tests to ensure your change is working in the correct way. Also be sure to change the version number in setup.py.

  2. Push updates to github repository

    3. When you push your changes to the remote repo, some tests will run and can be seen in the actions tab on our github site. Once you are positive everything is working well you can move on to publishing your changes.

  3. Update interface's copy of the repository to the latest version

    4. Our interface will not automatically use the latest version of the git repository. You must go onto cPanel and pull the changes your self. Scott Turro can help you gain access to this web hosting service. Once you are in cPanel go to the terminal under advanced and navigate to the repository cd tomo/public_html/cgi-bin/Quantum-Tomography_Git. Use git pull to update the code. If you encounter problems you can use git fetch to retrieve the updates and git status to compare the diffrences.

  4. Publish new version to pypi

    5. The last thing to do is release a new version to pypi. This can be done by creating a new release on the github page. A action titled Upload Python Package should run and update the next version. You can confirm this by looking at our pypi page or looking at the actions tab on github.

Maintain The Interface's Code

In the case that the tomography code is working fine but changes need to be made to the look of the interface, you will be making changes in a diffrent place. You will only need access to our cPanel for which you can contact gain access to our github repo, contact Scott Turro for access. Use fileManager to see our files and folders. The html is located in tomo/public_html. The php files and css files are most likely what you will be changing. For small changes I just copy the php file (say tomographyDemo.php) and edit the new copy. Once you are done editing and testing replace the original file with the edited copy. If you are looking to make big changes to the interface's layout and design, you should look into getting software to host the site locally. I used xampp but there is a variety of options.
